# 要件定義

@lus は、カップルが訪れた場所・思い出を地図上にピンで記録するFlutter Webアプリ。
サーバー・認証を持たず、すべてのデータをブラウザのローカルストレージ（IndexedDB）に保存する。

## 1. 機能要件

### F1: 地図表示
- flutter_map + OpenStreetMap タイルを使用。
- 初期表示は日本全体が見える位置・ズーム（中心：おおよそ緯度36.2, 経度138.2 付近）。
- 保存済みのピンをすべて地図上に表示する。
- ピンのタップで詳細画面へ遷移する。

### F2: ピン追加
- 地図のロングタップで、その座標を初期値としてピン追加フォームを開く。
- 入力項目：名称、種別（行った/予定/記念日）、提案者（自分/パートナー）、写真、★評価、コメント、訪問日。
- 保存するとローカルDB（pins テーブル）に追加され、地図に反映される。

### F3: ピン色分け
- ピンの種別に応じてマーカー色を変える。
  - 行った → ピンク
  - 予定 → ラベンダー
  - 記念日 → ゴールド

### F4: 提案者フラグ
- 各ピンに「誰が提案したか」を記録（自分 / パートナー）。
- 詳細・一覧で識別できるように表示する。

### F5: ピン詳細表示
- 1つのピンの全情報を表示：写真、★評価（1〜5）、コメント、訪問日、種別、提案者。
- 写真は image_picker で選択した画像を Base64 にエンコードして DB（`photo_base64`）に保存し表示する。
- 編集・削除への導線を持つ。

### F6: ピン編集・削除
- 既存ピンの全項目を編集できる。
- ピンを削除できる（削除前に確認）。

### F7: 一覧画面
- 全ピンをリスト表示する。
- 訪問日（日付）順にソートする。
- 各行から詳細画面へ遷移する。

### F8: 記念日カウントダウン
- anniversaries テーブルに登録された記念日までの残り日数を表示する。
- 当日・経過済みの扱いを明示する。

## 2. 非機能要件

### プラットフォーム
- v1 は Flutter Web のみ。将来的に iOS へ転用できる構成を保つ（プラットフォーム依存コードを局所化）。

### データ永続化
- drift（SQLite/WASM）を用い、ブラウザの IndexedDB に保存する。
- サーバー通信・外部同期は行わない。

### 認証・プライバシー
- 認証なし。データは端末ローカルにのみ保存される。
- 個人情報・写真は外部送信しない。

### 状態管理・ルーティング
- 状態管理は flutter_riverpod。
- 画面遷移は go_router。下部タブ（地図/一覧/記念日）と補助画面（詳細/編集）で構成する。

### オフライン
- 地図タイル取得を除き、アプリ機能はオフラインで動作する。

### 対象外（v1スコープ外）
- ローカル通知（Web 非対応のため v2 以降）。
- 複数端末間の同期、クラウドバックアップ。
- パートナー間のリアルタイム共有。

## 3. 開発方針
- 仮実装 → 動作確認 → デザイン調整 の順で進める。
- まず機能を最小構成で通し、その後 UI を整える。
