# 画面設計

v1 は 3 タブ + 2 補助画面の構成。下部タブで「地図 / 一覧 / 記念日」を切り替え、
ピン詳細・編集は補助画面として開く。画面遷移は go_router で管理する。

## 画面一覧

| ID | 画面 | 種別 | ルート | 役割 |
|----|------|------|--------|------|
| S1 | 地図画面（ホーム） | タブ | `/` | ピンの地図表示・追加の起点 |
| S2 | 一覧画面 | タブ | `/list` | 全ピンの日付順リスト |
| S3 | 記念日画面 | タブ | `/anniversaries` | 記念日の登録・カウントダウン |
| S4 | ピン詳細画面 | モーダル/別画面 | `/pin/:id` | 1ピンの詳細表示・編集/削除導線 |
| S5 | ピン編集・新規作成画面 | 別画面（共通） | `/pin/new`, `/pin/:id/edit` | ピンの追加・編集フォーム |

## タブ構成
- 下部ナビゲーションで S1 / S2 / S3 を切り替える。
- S4・S5 はタブの上に重なる形で開き、閉じると元のタブへ戻る。

## S1: 地図画面（ホーム・タブ）
- flutter_map で OpenStreetMap を表示。初期位置は日本中心。
- 保存済みピンを種別ごとの色（ピンク/ラベンダー/ゴールド）で表示。
- 操作：
  - ピンをタップ → S4（詳細）を開く。
  - 地図をロングタップ → その座標で S5（新規作成）を開く。

## S2: 一覧画面（タブ）
- 全ピンを訪問日（`visited_at`）順でリスト表示。各行：名称・種別・提案者・訪問日。
- 操作：行タップ → S4（詳細）を開く。

## S3: 記念日画面（タブ）
- anniversaries を一覧表示し、各記念日までの残り日数（カウントダウン）を表示。
- 操作：記念日の追加・編集・削除。

## S4: ピン詳細画面（モーダル/別画面）
- 表示：写真、名称、種別、提案者（自分/パートナー）、★評価、コメント、訪問日。
- 操作：
  - 編集 → S5（編集）を開く。
  - 削除 → 確認後に削除し、元のタブへ戻る。
  - 閉じる → 元の画面へ。

## S5: ピン編集・新規作成画面（共通）
- フォーム項目：名称、種別（行った/予定/記念日）、提案者（自分/パートナー）、写真（image_picker → Base64）、★評価、コメント、訪問日、座標。
- 新規時は S1 のロングタップ座標を初期値に設定。
- 操作：
  - 保存 → DB に反映し、新規は S4 もしくは元タブへ、編集は S4 へ戻る。
  - キャンセル → 変更を破棄して戻る。

## 遷移図

```
  ┌──────────── 下部タブ ────────────┐
  │  S1 地図   |   S2 一覧   |  S3 記念日 │
  └───┬──────────┬──────────────────┘
ロングタップ│ ピン/行タップ│
      ▼          ▼
  ┌──────────┐ ┌──────────┐
  │ S5 編集/新規 │ │ S4 ピン詳細 │
  └────┬─────┘ └──┬────┬──┘
       │保存       │編集 │削除
       └──► S4 ◄───┘    └─► 元タブ
```
