# コミット戦略

「仮実装 → 動作確認 → デザイン調整」の方針に沿い、v1 を 14 コミットで段階的に構築する。
各コミットは単体で動作確認できる粒度を目安とする。

| # | コミット | 内容 | 関連 |
|---|----------|------|------|
| 1 | chore: プロジェクト初期化 | Flutter Web プロジェクト生成、不要ファイル整理、`.gitignore` | 基盤 |
| 2 | chore: 依存追加 | flutter_riverpod / drift / flutter_map / image_picker / go_router を導入 | 基盤 |
| 3 | feat: ルーティング雛形 | go_router で下部タブ（地図/一覧/記念日）＋補助画面（詳細/編集）の空ルートを定義（S1〜S5） | S1-S5 |
| 4 | feat: DBスキーマ定義 | drift で pins / anniversaries テーブルを定義、IndexedDB 接続 | データ設計 |
| 5 | feat: 地図表示 | flutter_map + OSM、日本中心の初期表示 | F1 |
| 6 | feat: ピン読み込み表示 | DB のピンを地図にマーカー表示 | F1 |
| 7 | feat: ピン追加 | 地図ロングタップ → 編集画面で新規保存 | F2 |
| 8 | feat: ピン色分け | 種別ごとにマーカー色（ピンク/ラベンダー/ゴールド） | F3 |
| 9 | feat: 提案者フラグ | 提案者（自分/パートナー）の入力・保持・表示 | F4 |
| 10 | feat: ピン詳細表示 | 写真・★評価・コメント・訪問日を表示 | F5 |
| 11 | feat: ピン編集・削除 | 既存ピンの編集と削除（確認付き） | F6 |
| 12 | feat: 一覧画面 | 全ピンを日付順ソートでリスト表示 | F7 |
| 13 | feat: 記念日カウントダウン | anniversaries の残り日数表示 | F8 |
| 14 | style: デザイン調整 | 配色・余白・タイポグラフィなど UI 仕上げ | 全体 |

## 進め方
1. コミット 1〜4 で基盤（プロジェクト・依存・ルーティング・DB）を整える。
2. コミット 5〜13 で機能を 1 つずつ仮実装し、その都度動作確認する。
3. コミット 14 でまとめてデザイン調整を行う。

## コミットメッセージ規約
- プレフィックス：`feat` / `fix` / `chore` / `style` / `docs` / `refactor`。
- 1 コミット 1 目的を原則とする。
